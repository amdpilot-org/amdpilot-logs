  (256 experts) on ROCm with aiter enabled.

  Solution

  Added a fast path in fused_topk_bias that:

  1. Detects when aiter is enabled (rocm_aiter_ops.is_fused_moe_enabled() return
     rue) AND sigmoid scoring with bias correction is being used
  2. Computes num_expert_group using a helper function _compute_num_expert_group
     — finds the smallest divisor of num_experts such that num_experts / num_exp
     _group ≤ 32. For 256 experts → 8 groups (256/8=32)
  3. Sets topk_group = num_expert_group (semantically equivalent to global top-k
     th bias)
  4. Routes to rocm_aiter_grouped_topk() which calls aiter's optimized biased_gr
     ed_topk kernel

  outer/fused_topk_bias_router.py
