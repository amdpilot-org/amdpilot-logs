## Task Description

# [AMD] Enable cudagraph for aiter NSA backend and add aiter impl for NSA prefill

## Bug Description

The Native Sparse Attention (NSA) decode path in SGLang's aiter backend uses a boolean mask to extract valid KV indices from the page table:

```python
kv_indices = page_table_1[page_table_1 != -1]
```

This boolean mask produces a **dynamic output shape** that cannot be captured into a CUDA graph. As a result, CUDA graph capture fails for NSA decode when using the aiter backend, forcing fallback to eager execution and degrading throughput significantly.

Additionally, the NSA prefill phase lacks an aiter backend implementation entirely, requiring the use of the slower tilelang backend.

## Expected Fix

1. **NSA Decode cudagraph fix**: Replace the dynamic boolean mask with a deterministic triton kernel (`get_valid_kv_indices`) that extracts valid KV indices from the transformed page table without producing dynamic shapes. This allows CUDA graph capture to succeed.

2. **NSA Prefill aiter implementation**: Add a simple aiter-based implementation for the NSA prefill phase, complementing the existing decode path.

The key files to modify are:
- `python/sglang/srt/layers/attention/nsa/triton_kernel.py` — Add the `get_valid_kv_indices` triton kernel
- `python/sglang/srt/layers/attention/nsa_backend.py` — Integrate the new kernel and add aiter prefill support

## Environment

- **Hardware**: MI355X x8
- **Repository**: SGLang at `/sgl-workspace/sglang`
- **Key file(s)**: `python/sglang/srt/layers/attention/nsa/triton_kernel.py`, `python/sglang/srt/layers/attention/nsa_backend.py`

python3 -m sglang.launch_server --model /models/DeepSeek-V3.2-Exp \

    --nsa-prefill-backend aiter \
